# 大屏功能方案文档

## 1. 需求分析

### 1.1 功能概述
为股票策略系统开发一个数据可视化大屏，实时展示股票行情、策略执行、业绩指标和系统监控数据，适用于展示和监控场景。

### 1.2 核心需求

| 模块 | 功能描述 |
|------|----------|
| 策略执行数据 | 买卖信号、策略执行状态、持仓情况、交易记录 |
| 业绩指标 | 累计收益、收益曲线、胜率、最大回撤、风险指标 |
| 系统监控 | API状态、数据同步状态、告警信息、系统健康度 |

### 1.3 用户场景
- 投资决策监控：实时查看策略执行情况
- 团队展示汇报：大屏展示整体业绩表现
- 系统运维监控：监控系统运行状态

## 2. 数据库设计

### 2.1 表结构设计

#### 2.1.1 策略执行表 (strategy_executions)
```sql
CREATE TABLE strategy_executions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    strategy_id INTEGER NOT NULL,              -- 策略ID
    strategy_name VARCHAR(100),                -- 策略名称
    stock_code VARCHAR(20),                    -- 股票代码
    signal_type VARCHAR(10),                   -- 信号类型: BUY/SELL/HOLD
    execution_status VARCHAR(20),              -- 执行状态: PENDING/EXECUTED/FAILED
    position_size DECIMAL(10,3),               -- 持仓数量
    position_cost DECIMAL(15,2),               -- 持仓成本
    target_price DECIMAL(10,3),                -- 目标价格
    stop_loss DECIMAL(10,3),                   -- 止损价格
    executed_at TIMESTAMP,                     -- 执行时间
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_strategy_id ON strategy_executions(strategy_id);
CREATE INDEX idx_stock_code ON strategy_executions(stock_code);
CREATE INDEX idx_execution_status ON strategy_executions(execution_status);
```

#### 2.1.2 持仓表 (positions)
```sql
CREATE TABLE positions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    stock_code VARCHAR(20) NOT NULL,
    stock_name VARCHAR(50),
    position_size DECIMAL(10,3),               -- 持仓数量
    cost_price DECIMAL(10,3),                  -- 成本价
    current_price DECIMAL(10,3),               -- 当前价
    market_value DECIMAL(15,2),                -- 市值
    profit_loss DECIMAL(15,2),                 -- 盈亏
    profit_percent DECIMAL(6,4),               -- 盈亏比例
    update_time TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_stock_code ON positions(stock_code);
```

#### 2.1.3 业绩指标表 (performance_metrics)
```sql
CREATE TABLE performance_metrics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date DATE NOT NULL,                        -- 日期
    total_return DECIMAL(10,4),                -- 累计收益率
    daily_return DECIMAL(6,4),                 -- 日收益率
    win_rate DECIMAL(6,4),                     -- 胜率
    max_drawdown DECIMAL(6,4),                 -- 最大回撤
    sharpe_ratio DECIMAL(8,4),                 -- 夏普比率
    volatility DECIMAL(6,4),                   -- 波动率
    total_trades INTEGER,                      -- 总交易次数
    profit_trades INTEGER,                     -- 盈利交易次数
    loss_trades INTEGER,                       -- 亏损交易次数
    avg_profit DECIMAL(10,3),                  -- 平均盈利
    avg_loss DECIMAL(10,3),                    -- 平均亏损
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_date ON performance_metrics(date);
```

#### 2.1.4 系统监控表 (system_monitor)
```sql
CREATE TABLE system_monitor (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    monitor_type VARCHAR(50),                  -- 监控类型: API/DATA/SYSTEM
    monitor_name VARCHAR(100),                 -- 监控项名称
    status VARCHAR(20),                        -- 状态: HEALTHY/WARNING/ERROR
    value DECIMAL(10,2),                       -- 监控值
    threshold DECIMAL(10,2),                   -- 阈值
    message TEXT,                              -- 消息
    update_time TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_monitor_type ON system_monitor(monitor_type);
CREATE INDEX idx_status ON system_monitor(status);
```

#### 2.1.5 告警表 (alerts)
```sql
CREATE TABLE alerts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    alert_type VARCHAR(50),                    -- 告警类型
    alert_level VARCHAR(20),                   -- 告警级别: INFO/WARNING/ERROR/CRITICAL
    title VARCHAR(200),                        -- 告警标题
    content TEXT,                              -- 告警内容
    source VARCHAR(100),                       -- 来源
    is_resolved BOOLEAN DEFAULT FALSE,         -- 是否已解决
    resolved_at TIMESTAMP,                     -- 解决时间
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_alert_level ON alerts(alert_level);
CREATE INDEX idx_is_resolved ON alerts(is_resolved);
CREATE INDEX idx_created_at ON alerts(created_at);
```

## 3. 技术栈

### 3.1 前端技术栈（大屏专用）
| 技术 | 用途 | 说明 |
|------|------|------|
| Vue3 + TypeScript | 核心框架 | 组合式API |
| ECharts | 可视化图表 | 大屏图表首选 |
| DataV | 大屏组件库 | DataV Vue3版（若可用） |
| UnoCSS | 原子化CSS | 响应式布局 |
| Axios | HTTP请求 | 数据获取 |
| Vue Router | 路由 | 页面切换 |

### 3.2 后端技术栈
| 技术 | 用途 | 说明 |
|------|------|------|
| FastAPI | 后端框架 | 异步高性能 |
| Tortoise ORM | 数据库ORM | 异步操作 |
| SQLite | 数据库 | 轻量存储 |
| Pydantic | 数据验证 | Schema定义 |

### 3.3 大屏设计规范
| 参数 | 值 | 说明 |
|------|-----|------|
| 分辨率 | 1920x1080 | 标准大屏 |
| 背景色 | 深蓝渐变 | #0a1a3a → #1a2a4a |
| 字体 | 思源黑体 | 清晰易读 |
| 动画 | 3-5秒刷新 | 数据更新动画 |

## 4. 接口设计

### 4.1 API列表

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取策略执行 | GET | /api/dashboard/strategies | 策略执行状态 |
| 获取持仓列表 | GET | /api/dashboard/positions | 当前持仓 |
| 获取业绩指标 | GET | /api/dashboard/performance | 业绩统计 |
| 获取收益曲线 | GET | /api/dashboard/performance/chart | 收益曲线数据 |
| 获取系统状态 | GET | /api/dashboard/system/status | 系统监控状态 |
