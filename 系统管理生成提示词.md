

# 用户权限机制设计文档

本文档描述管理系统的系统管理扩展功能.包括基础设施配置、权限颗粒度选项、各功能模块的详细说明及代码生成指南。

此文档开发受AGENTS.md 文件约束，必须遵循，执行任务前先读取AGENTS.md。

## 1. 基础设施配置

### 1.1 Redis缓存 (可选)

| 配置项 | 功能说明 | 技术实现 |
|--------|----------|----------|
| 字典缓存 | 缓存系统字典数据，减少数据库查询 | Redis Hash存储，Key: `dict:{type}` |
| 配置缓存 | 缓存系统参数配置，支持动态更新 | Redis String存储，Key: `config:{key}` |
| Session存储 | 用户登录状态管理，支持分布式部署 | Redis存储Session，支持Token刷新 |
| 限流控制 | API访问频率限制，防止恶意请求 | Redis计数器，滑动窗口算法 |

**选择Redis**: 支持分布式部署，缓存数据一致性更好
**不选择Redis**: 使用内存缓存(TTL缓存)，仅支持单机部署

## 2. 权限颗粒度配置

系统支持三种权限颗粒度，可按需选择组合：

### 2.1 页面权限

控制用户能访问哪些页面/菜单，实现动态路由。

| 功能点 | 说明 |
|--------|------|
| 菜单过滤 | 根据用户角色关联的菜单，动态生成可访问菜单列表 |
| 路由控制 | 前端根据菜单数据动态注册路由，无权限页面不可访问 |
| 菜单隐藏 | 未授权菜单自动隐藏，用户无法看到 |

**数据影响**: 需要角色表、角色菜单关联表

### 2.2 按钮权限

控制用户能看到/操作哪些按钮，实现精细化权限控制。

| 功能点 | 说明 |
|--------|------|
| 按钮显示控制 | 通过 `v-permission` 指令控制按钮显示/隐藏 |
| 权限标识定义 | 每个按钮定义唯一权限标识，如 `user:add`、`user:edit` |
| API权限校验 | 后端API通过权限装饰器校验用户操作权限 |

**数据影响**: 菜单表增加 `permission`、`api_url`、`api_method` 字段

### 2.3 数据权限

控制用户能看到哪些数据范围，实现数据隔离。

## 3. 数据权限范围

| data_scope | 含义 | SQL过滤逻辑 | 适用场景 |
|------------|------|-------------|----------|
| 1 | 全部数据 | 无过滤条件 | 系统管理员、超级管理员 |
| 2 | 自定义部门 | `dept_id IN (指定部门列表)` | 跨部门管理人员 |
| 3 | 本部门数据 | `dept_id = 用户部门ID` | 部门经理 |
| 4 | 本部门及下级 | `dept_id IN (本部门及所有子部门)` | 上级管理人员 |
| 5 | 仅本人数据 | `create_by = 用户ID` | 普通员工 |

**数据影响**: 角色表增加 `data_scope` 字段，需要角色部门关联表

**Redis缓存说明**:
- 如果选择Redis: 字典、配置使用Redis缓存，支持分布式部署
- 如果不选择: 字典、配置使用内存缓存(TTL缓存)，仅支持单机部署

**权限颗粒度说明**:
- **页面权限**: 菜单表增加权限标识，角色关联菜单，前端根据用户菜单动态生成路由
- **按钮权限**: 按钮定义权限标识，前端使用 `v-permission` 指令控制显示
- **数据权限**: 角色配置数据范围，后端查询时自动拼接数据过滤条件

**根据选择生成对应的菜单数据和代码**

### Step 1: 需求分析
- 确认业务模块名称
- 确认功能需求 (CRUD、导入导出、权限控制等)
- 确认特殊字段和业务规则

### Step 2: 数据库设计
- 设计表结构，遵循命名规范
- 生成建表SQL语句
- 创建Tortoise-ORM模型

### Step 3: 后端代码生成
按顺序生成:
1. `models/{module}.py` - 数据模型
2. `schemas/{module}.py` - Pydantic模型
3. `route/{module}.py` - 路由指向
4. `services/{module}.py` - 业务逻辑
5. `api/v1/{module}.py` - API路由

### Step 4: 前端代码生成
按顺序生成:
1. `types/{module}.ts` - TypeScript类型
2. `api/{module}.ts` - API接口
3. `views/{module}/index.vue` - 列表页面
4. `views/{module}/components/Dialog.vue` - 弹窗组件

### Step 5: 质量验证
- 检查代码是否符合规范
- 确认所有文件命名正确
- 确认分层架构正确

### 后端 - 权限服务

```python
# services/permission.py
from typing import List, Set, Optional
from models.user import User
from models.role import Role, UserRole, RoleMenu, RoleDept
from models.menu import Menu
from tortoise.expressions import Q


class PermissionService:
    """权限服务层"""

    async def get_user_menus(self, user_id: int) -> List[dict]:
        """获取用户菜单列表 (页面权限)"""
        role_ids = await self.get_user_role_ids(user_id)
        if not role_ids:
            return []

        menu_ids = await RoleMenu.filter(role_id__in=role_ids).values_list('menu_id', flat=True)
        menus = await Menu.filter(
            id__in=menu_ids,
            status=0,
            menu_type__in=['catalog', 'menu']
        ).order_by('sort')

        return self._build_menu_tree(menus)

    async def get_user_permissions(self, user_id: int) -> Set[str]:
        """获取用户所有权限标识 (按钮权限)"""
        role_ids = await self.get_user_role_ids(user_id)
        if not role_ids:
            return set()

        menu_ids = await RoleMenu.filter(role_id__in=role_ids).values_list('menu_id', flat=True)
        permissions = await Menu.filter(
            id__in=menu_ids,
            menu_type='button',
            permission__isnull=False
        ).values_list('permission', flat=True)

        return set(permissions)

    async def check_permission(self, user_id: int, permission: str) -> bool:
        """检查用户是否有某个权限"""
        permissions = await self.get_user_permissions(user_id)
        return permission in permissions

    async def get_data_scope_filter(self, user_id: int, table_alias: str = None) -> Optional[Q]:
        """获取数据权限过滤条件"""
        user_roles = await UserRole.filter(user_id=user_id).prefetch_related('role')
        if not user_roles:
            return Q(id__in=[])

        data_scope = min(ur.role.data_scope for ur in user_roles if ur.role)
        user = await User.get_or_none(id=user_id)
        if not user:
            return Q(id__in=[])

        alias = f"{table_alias}__" if table_alias else ""

        if data_scope == 1:
            return None  # 全部数据
        elif data_scope == 2:
            # 自定义部门
            role_ids = [ur.role_id for ur in user_roles]
            dept_ids = await RoleDept.filter(role_id__in=role_ids).values_list('dept_id', flat=True)
            return Q(**{f"{alias}dept_id__in": dept_ids})
        elif data_scope == 3:
            # 本部门
            return Q(**{f"{alias}dept_id": user.dept_id})
        elif data_scope == 4:
            # 本部门及下级
            dept_ids = await self._get_dept_and_children(user.dept_id)
            return Q(**{f"{alias}dept_id__in": dept_ids})
        elif data_scope == 5:
            # 仅本人
            return Q(**{f"{alias}create_by": user_id})
        return None

    async def get_user_role_ids(self, user_id: int) -> List[int]:
        return await UserRole.filter(user_id=user_id).values_list('role_id', flat=True)

    def _build_menu_tree(self, menus: List[Menu], parent_id: int = 0) -> List[dict]:
        tree = []
        for menu in menus:
            if menu.parent_id == parent_id:
                node = {
                    'id': menu.id,
                    'menu_name': menu.menu_name,
                    'menu_type': menu.menu_type,
                    'menu_icon': menu.menu_icon,
                    'menu_path': menu.menu_path,
                    'sort': menu.sort,
                    'children': self._build_menu_tree(menus, menu.id)
                }
                tree.append(node)
        return sorted(tree, key=lambda x: x['sort'])
```

# api/deps.py
from fastapi import Depends, HTTPException
from fastapi.security import OAuth2PasswordBearer
from jose import jwt
from core.config import settings
from models.user import User
from services.permission import PermissionService

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/api/v1/auth/login")


async def get_current_user(token: str = Depends(oauth2_scheme)) -> User:
    """获取当前用户"""
    try:
        payload = jwt.decode(token, settings.SECRET_KEY, algorithms=[settings.ALGORITHM])
        user_id = payload.get("sub")
        if not user_id:
            raise HTTPException(status_code=401, detail="无效的认证凭证")
    except Exception:
        raise HTTPException(status_code=401, detail="无效的认证凭证")

    user = await User.get_or_none(id=int(user_id))
    if not user:
        raise HTTPException(status_code=401, detail="用户不存在")
    return user


def check_permission(permission: str):
    """权限检查装饰器"""
    async def permission_checker(current_user: User = Depends(get_current_user)):
        perm_service = PermissionService()
        has_permission = await perm_service.check_permission(current_user.id, permission)
        if not has_permission:
            raise HTTPException(status_code=403, detail="无权限访问")
        return current_user
    return Depends(permission_checker)

### 前端 - 权限状态管理

```typescript
// stores/permission.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { getUserMenus, getUserPermissions } from '@/api/permission'

export const usePermissionStore = defineStore('permission', () => {
  const menus = ref<any[]>([])
  const permissions = ref<Set<string>>(new Set())

  const hasPermission = computed(() => (permission: string) => {
    return permissions.value.has(permission)
  })

  const hasSomePermission = computed(() => (perms: string[]) => {
    return perms.some(p => permissions.value.has(p))
  })

  const init = async () => {
    const [menusRes, permsRes] = await Promise.all([
      getUserMenus(),
      getUserPermissions()
    ])
    menus.value = menusRes.data
    permissions.value = new Set(permsRes.data)
  }

  return { menus, permissions, hasPermission, hasSomePermission, init }
})
```

### 前端 - 权限指令

```typescript
// directives/permission.ts
import type { Directive, DirectiveBinding } from 'vue'
import { usePermissionStore } from '@/stores/permission'

/**
 * 权限指令
 * 用法: v-permission="'user:add'" 或 v-permission="['user:add', 'user:edit']"
 */
export const permission: Directive = {
  mounted(el: HTMLElement, binding: DirectiveBinding<string | string[]>) {
    const permissionStore = usePermissionStore()
    const value = binding.value
    if (!value) return

    let hasPermission = false
    if (typeof value === 'string') {
      hasPermission = permissionStore.hasPermission(value)
    } else if (Array.isArray(value)) {
      hasPermission = permissionStore.hasSomePermission(value)
    }

    if (!hasPermission) {
      el.parentNode?.removeChild(el)
    }
  }
}

export function setupPermissionDirective(app: App) {
  app.directive('permission', permission)
}
```

    component VARCHAR(200) COMMENT '组件路径',
    status TINYINT NOT NULL DEFAULT 0 COMMENT '状态: 0=启用, 1=禁用',
    redirect VARCHAR(200) COMMENT '重定向路径',
    INDEX idx_menu_parent (parent_id),
    INDEX idx_menu_sort (sort)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统菜单表';
